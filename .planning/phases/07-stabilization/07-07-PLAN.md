# Plan: 07-07 — SEO & Meta Tags Hardening

## Goal
Make job listings and pages crawlable and shareable. A job board without SEO is invisible.

## Proposed Changes

### 1. Global Meta Tags
- Add `<meta name="description">` to `index.html` with platform-level description.
- Add Open Graph tags (`og:title`, `og:description`, `og:image`, `og:url`, `og:type`).
- Add Twitter Card tags (`twitter:card`, `twitter:title`, `twitter:description`).
- Add `<link rel="canonical">` to prevent duplicate content issues.

### 2. Dynamic Per-Page Titles
- Install `react-helmet-async` (or equivalent).
- Add `<Helmet>` to key route components:
  - Job listing page: `{job.title} at {company} | IEH`
  - Company profile page: `{company.name} — Careers | IEH`
  - Seeker dashboard: `Dashboard | IEH`
  - Referral hub: `Referral Hub | IEH`

### 3. Static Assets
- Create `public/robots.txt` allowing all crawlers with sitemap reference.
- Create `public/sitemap.xml` with static routes (dynamic job routes are out of scope for SPA without SSR).
- Add proper favicon set (16x16, 32x32, apple-touch-icon, `manifest.json`).

### 4. Structured Data (JSON-LD)
- Add `JobPosting` schema markup on job detail pages (Google for Jobs integration).
- Add `Organization` schema on the landing page.

## Verification Plan
1. Run Google Rich Results Test on a job detail URL — passes.
2. Run Lighthouse SEO audit — score ≥ 90.
3. Verify OG tags render correctly in social media preview debuggers (Facebook, Twitter, LinkedIn).
