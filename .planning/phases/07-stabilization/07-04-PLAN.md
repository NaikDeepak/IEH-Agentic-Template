# Plan: 07-04 — Performance & Accessibility Audit

## Goal
Achieve Lighthouse Performance ≥ 90 and WCAG AA compliance across all key pages.

## Proposed Changes

### 1. Lighthouse Baseline
- Run Lighthouse CI on: Landing page, Job listings, Seeker dashboard, Employer dashboard, Referral hub.
- Document baseline scores (Performance, Accessibility, SEO, Best Practices).

### 2. Performance Fixes
- Audit and fix lazy-loading gaps (heavy components: `recharts`, `jspdf`, `html2canvas`).
- Optimize image delivery: WebP format, responsive `srcset`, lazy `loading` attribute.
- Review bundle splitting: ensure Vite is code-splitting by route.
- Add `rel="preconnect"` for Firebase, Sentry, and Google Fonts.

### 3. Accessibility Fixes
- Audit all form inputs for labels, `aria-label`, and `aria-describedby`.
- Verify color contrast ratios (WCAG AA: 4.5:1 for normal text, 3:1 for large).
- Add skip-to-content link on all pages.
- Ensure all interactive elements are keyboard-navigable.
- Add focus management for modals and overlays.

### 4. Mobile Responsiveness
- Test all pages at 375px (iPhone SE), 390px (iPhone 14), and 768px (tablet).
- Fix any layout breaks, overflow issues, or touch target size violations (minimum 44x44px).

## Verification Plan
1. Lighthouse CI scores ≥ 90 on all 5 key pages.
2. axe-core accessibility scan returns 0 critical violations.
3. Visual verification on mobile viewport screenshots.
